《Designing Data Intensive Applications》读书笔记 - 数据库复制

这一章讲数据库复制（Replication），目标很简单就是保存数据副本在多个机器上，但是实现却没那么容易。首先需要数据复制的几个原因：

  1. 数据中心地理上更靠近用户
  2. 增强可用性，即便部分服务器节点失败，整个系统依然可用
  3. 提高读取的吞吐量
